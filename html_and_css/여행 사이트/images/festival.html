  1. 위 사진과 같이 images/hero_image.jpg 파일로 .hero-header의 배경 이미지를 설정해주고, .learn-more 버튼을 스타일링 해주세요.

BEST 답변

태그들이 가지는 기본적인 스타일에 대해서 이해하시면 더 도움이 될 것 같은데요.

h1 이나 a 태그 처럼 직접적으로 아무런 스타일을 주지 않아도 기본적으로 스타일을 가진 태그들이 있습니다.

브라우저별로 미세하게 차이가 있지만,

h1 태그는 기본적으로

h1{ 
  display: ~; 
  font-size: ~ ; 
  font-weight: ~ ; 
  margin: ~ ; 
}

4가지 속성을 가지고 있어요. 그래서

block의 성질(display)을 가지고,

더 크고(font-size),

두껍고(font-weight),

태그들 사이에 적당한 여백(margin)을 가지고 있죠.

a 태그는 어떨까요?

a태그는 기본적으로

a{ 
  color: ~ ; 
  text-decoration : ~ ; 
  cursor: ~ ; 
}

3가지 속성을 기본적으로 가지고 있습니다. 그래서

파란색 글자(color)와 ,

글자 아래 밑줄(text-decoration﻿)과,

마우스를 올리면 손가락으로 변하는 효과(cursor)를 갖고 있는거죠.

성준님이 궁금해하는 가장 큰 포인트가 바로 이 부분 이에요.

h1태그와 같은 경우에는 기본적으로 가지고 있는 color 값이 없기 때문에

자연스럽게 부모태그에 있는 color 값을 내려받게 되고,

a 태그는 이미 color 값을 가지고 있기 때문에

원하는 색이 있을 경우에 반드시 별도로 지정해줘야 하는 걸로 이해하시면 좋을것 같아요.

따라서 a 태그가 부모태그의 color 값을 내려받기를 원하신다면

a{ 
  color: inherit;
}

이렇게 값을 주면 되지 않을까요? 이해가 되셨는지 모르겠지만, 조금이나마 도움이 되었으면 좋겠습니다 :)

BEST 답변

.info 가 inline-block & width: 100%이기 때문이에요.

태그들의 속성이 단순히 block 일 경우에는 띄어쓰기나 줄바꿈이 출력에 영향을 미치지 않을 수 있지만 inline의 성격이 들어갈 경우에는 글자처럼 인식이 되기 때문에, 코드속의 개행과 띄어쓰기에 대해서 띄어쓰기 한 칸만큼의 영향을 받아요 :)

    <div class="helper"></div>
    <div class="info">
      <h1>10th Summer Music Festival</h1>
      <h2>20 - 21 August, 2017</h2>

위 코드에서 개행이 띄어쓰기 한 칸의 영향을 미치기 때문에 .header-hero 입장에서는 helper 태그 다음에 띄어쓰기를 하나 그려줘야 되요 width가 없는 helper의 경우에는 너비의 공간을 차지하지 않지만 띄어쓰기 한칸의 경우에는 폰트사이즈를 기준으로 띄어쓰기 한칸만큼의 공간을 그려주게되어 작은 공간을 차지하게 되요. 그리도나서 width가 100% 인 info를 그려야 되는데 띄어쓰기 한칸의 작은 공간에 밀려서 어쩔수 없이 width 100%를 맞추기 위해 그 다음줄에 info를 그리게 된 거에요 :)

아래 코드 처럼 굳이 개행이 아니라 사이에 띄어쓰기 하나만 주어도 같은 결과가 나올 거에요.

    <div class="helper"></div> <div class="info">
      <h1>10th Summer Music Festival</h1>
      <h2>20 - 21 August, 2017</h2>

다른 확인 방법으로는 info에 width를 99%(혹은 그 이하)로 적용한다음 개발자 도구에서 info의 범위를 확인해보면 왼쪽에 띄어쓰기 한칸만큼 여백이 더 들어간 걸 확인할 수 있을 거에요! :)
